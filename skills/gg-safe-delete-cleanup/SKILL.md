# GG Safe Delete / Cleanup Skill

Use this skill for cleanup, deletion, pruning, and deprecation.

## Prime Directive

Do not delete code just because it looks unused. Prove ownership, usage, replacement, and guard impact first.

## Delete Classification

Classify candidate:

- `generated artifact`: can be regenerated by a tool.
- `dead source`: no import/reference/runtime use.
- `legacy source`: old path kept for transition.
- `protected contract`: must not delete without explicit architecture task.
- `buyer clutter`: internal-only file that should be excluded from buyer package, not deleted from dev repo.

## Usage Proof

Before deletion, search:

- imports;
- script references;
- workflow references;
- docs references;
- registry references;
- runtime hooks;
- data attributes;
- CSS selectors;
- Blogger template references;
- Worker routing references.

Do not rely only on filename search. Search semantic hooks like `data-gg-*`, classes, IDs, copy keys, route keys, and command names.

## Cleanup Procedure

1. Inventory candidates.
2. Classify each candidate.
3. Record proof.
4. Remove one group at a time.
5. Update references/docs/scripts.
6. Run focused checks.
7. If generated, verify regeneration path.
8. Handoff removed files and why.

## Buyer Package Cleanup

If the goal is buyer packaging, prefer exclude rules over deletion for dev-only assets.

Dev-only examples:

- internal QA reports;
- task prompts;
- experiments;
- archives;
- snapshots;
- debug logs;
- old zips.

## Protected Areas

Do not delete without explicit task:

- Blogger comments plumbing;
- Blog1 detail rendering;
- root/store source boundary registry;
- preview sheet controller;
- discovery taxonomy;
- theme system;
- Worker static routing;
- template fingerprint tool;
- source-of-truth docs referenced by guards.

## Validation

```bash
git diff --check
npm run ci:qa
```

If assets/store/worker/template changed, run related focused checks.

## Handoff Format

Report:

- deleted files;
- reason/proof for each group;
- references updated;
- commands run;
- risks and rollback path.
