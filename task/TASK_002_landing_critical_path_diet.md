# TASK 002 — Landing critical path diet

## Context

`/landing` is the identity/home surface and may be the first page reached by users, search, AI, social, or direct links.

It must be readable and fast before app-like enhancements load.

The site may still be in development/staging/performanceAudit mode. Do not switch production.

## Goal

Reduce `/landing` critical-path cost without changing the route truth or visual identity.

Target first improvement:

```text
/landing mobile Lighthouse should improve meaningfully from current baseline.
Aim first for >=65, then stabilize toward >=80 after subsequent tasks.
```

## Required changes

### 1. Preserve Material Symbols subset only

Because Material Symbols are intentionally retained:

```text
- Do not remove Material Symbols entirely.
- Do remove full/unbounded Material Symbols loads.
- Landing may only load route-specific subset generated by config/icons.config.json.
- No variable axis range unless proven necessary.
```

If landing currently loads full Material Symbols in head, replace with subset URL from TASK 001.

### 2. Disable expensive mobile effects on first paint

Add or merge a mobile-first landing override:

```css
@media (max-width: 768px) {
  body {
    background: var(--gg-surface-page) !important;
  }

  .gg-topbar,
  .gg-dock,
  .gg-detail-outline,
  .gg-sheet__scrim {
    -webkit-backdrop-filter: none !important;
    backdrop-filter: none !important;
  }

  .gg-topbar,
  .gg-dock,
  .gg-detail-outline,
  .gg-card,
  .gg-sheet__panel {
    box-shadow: none !important;
  }

  .gg-detail-outline,
  .gg-dock,
  .gg-reveal {
    will-change: auto !important;
  }

  .gg-reveal {
    opacity: 1 !important;
    transform: none !important;
    transition: none !important;
  }
}
```

Do not destroy desktop design. This is mainly mobile first-paint discipline.

### 3. Do not hide content waiting for JS

Initial landing sections must be visible without JavaScript.

Preferred:

```css
.gg-reveal {
  opacity: 1;
  transform: none;
  transition: none;
}
```

If reveal animation is needed later, it must run after idle or user intent, not before first paint.

### 4. Defer non-critical landing JavaScript

Immediate JS may do only:

```text
- theme preboot
- basic route/body state
- basic link/button behavior if necessary
```

Defer:

```text
- outline scrollspy
- dock hide-on-scroll logic
- command/search result rendering
- language/theme panel UI
- sheet gesture physics
- QA/debug matrices
- large copy registry hydration
```

Use helper:

```js
function deferLandingEnhancement(callback, timeout) {
  var wait = typeof timeout === "number" ? timeout : 1200;
  if ("requestIdleCallback" in window) {
    window.requestIdleCallback(callback, { timeout: wait });
  } else {
    window.setTimeout(callback, Math.min(wait, 800));
  }
}
```

Then:

```js
deferLandingEnhancement(function () {
  // outline, panels, scroll state, language/theme, command search
});
```

### 5. Keep route/canonical truth

Do not alter:

```text
/landing public route
/landing.html internal asset behavior unless existing Worker task requires route hygiene
Home => /landing
Blog => /
```

## Acceptance criteria

```text
- Landing content is readable without JS.
- No full/unbounded Material Symbols URL appears on /landing.
- Mobile first paint does not depend on reveal JS.
- Mobile first paint avoids backdrop-filter and heavy shadows.
- Non-critical landing controllers run after first paint/idle.
- Accessibility remains >=95.
- /landing route/canonical truth remains unchanged.
```
