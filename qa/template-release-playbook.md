# Blogger Template Manual Release Checklist

Source of truth:
- `index.prod.xml`

Publish artifact:
- `dist/blogger-template.publish.xml` (generated by `npm run gaga:template:pack`)

When manual Blogger publish is required:
- Whenever `node qa/template-fingerprint.mjs --value` changes.
- Whenever SSR/template markup changes in `index.prod.xml` (route/head/shell rows/panel markup).

Manual publish action (Blogger):
1. Run `npm run gaga:template:pack`.
2. Open Blogger Admin for `www.pakrpp.com`.
3. Go to `Theme` -> `Edit HTML`.
4. Replace template HTML with `dist/blogger-template.publish.xml`.
5. Save/publish the template.

Mandatory post-publish verification (public production):
1. `npm run gaga:template:status -- https://www.pakrpp.com`
2. `npm run gaga:template:proof -- https://www.pakrpp.com`
3. `npm run gaga:verify-template`

Release is complete only if:
- Template status reports `state=MATCH`.
- Template proof reports `TEMPLATE PROOF RESULT=PASS`.
- Live smoke passes with release state `blogger_template_parity_verified`.
- `/` and `/landing` canonical/surface checks pass.
- `/blog` redirects to `/` (legacy redirect only).
- Empty editorial shell/rows are not visible in SSR output.

Release state wording:
- `worker_assets_deployed_only__blogger_template_publish_required` -> Worker/assets deployed only; Blogger template publish required.
- `blogger_template_publish_completed_but_drift_remains` -> publish done, but live template still not matching repo.
- `blogger_template_parity_verified` -> template publish + live parity verified.

If any verification fails:
- Treat release as incomplete.
