# Blogger Template Manual Release Checklist

Template source of truth:
- `index.prod.xml`

Publish artifact:
- `dist/blogger-template.publish.xml` (generated by `npm run gaga:template:pack`)

## Important scope truth

- `npm run gaga` now means GitHub-first code shipping (commit + push), not local Wrangler deploy.
- Cloudflare deploy workflow publishes Worker/assets only.
- Blogger template/theme publish is still manual.

## When manual Blogger publish is required

- whenever `index.prod.xml` changes
- whenever `node qa/template-fingerprint.mjs --value` changes
- whenever template-coupled SSR markup changes (layout shell, route markers, editorials, detail/listing contracts)

## Manual Blogger publish steps

1. Run `npm run gaga:template:pack`.
2. Open Blogger Admin for `www.pakrpp.com`.
3. Go to `Theme` -> `Edit HTML`.
4. Replace template HTML with `dist/blogger-template.publish.xml`.
5. Save/publish the template.

## Mandatory post-publish verification

1. `npm run gaga:template:status -- https://www.pakrpp.com`
2. `npm run gaga:template:proof -- https://www.pakrpp.com`
3. `npm run gaga:verify-template`

Release is complete only if:
- template status is `state=MATCH`
- template proof is `TEMPLATE PROOF RESULT=PASS`
- full live smoke passes (`LIVE SMOKE RESULT: PASS`)

## When to run strict full public smoke

Use strict lane when template parity must be enforced as hard gate:
- GitHub Actions `workflow_dispatch` with `strict_template_parity=true`
- local strict run via `npm run gaga:verify-template`

If Worker release is green while template changed, treat it as:
- `worker_assets_deployed_only__blogger_template_publish_required`
- manual Blogger publish is still pending
