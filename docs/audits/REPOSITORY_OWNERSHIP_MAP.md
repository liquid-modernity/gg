## Duplicate or Unclear Ownership

- Root copy JSON aliases and `registry/copy/*` look duplicated, but current guards and static staging expect both. Treat consolidation as high-risk until a copy-registry task proves one canonical runtime path.
- `src/knowledge base` and `assets/knowledge base` use spaces and mirror route assets. Renaming would improve shell ergonomics but requires build, guard, and public asset reference updates.
- Root reports are partly documentation, partly guard fixtures. Moving them into `docs/reports/` is attractive but not low risk because several guards and docs reference exact root paths.
- `registry/original/*` and `docs/archive/template-deprecated/*` are historical, but they still carry architecture/handoff value. Do not delete or move without an archive policy task.
- `schema-jsonld-guard.mjs` exists at root while the canonical guarded script also exists under `qa/`. Treat root ownership as unclear until usage proof.
