# Repository Restructure Risk Register

Scope: planning-only risk register. Recommendations here are future-task guidance, not permission to change files in this task.

|---|---|---|---|---|---|

| Move `src/registry/*` | HIGH / DO NOT MOVE | Route truth, Store source boundary, discovery/action/dock registries, Store feed source config | Store tools, app source, docs, guards | Content source boundary, registry, Store proof, CI | Do not move |
| Move `src/js/modules/*` | HIGH / DO NOT MOVE | Controller/core/service ownership map, JS fragment references, agent source mapping | Controller inventory, docs, guards, any future packer | Controller core/adapters, shell, discovery, preview, comments guards | Do not move until JS source model is redesigned |
| Move `src/css/modules/*` | HIGH / DO NOT MOVE | Generated mirrors, wired module source, CSS guard parity, manual/advisory CSS maps | CSS sync tool, CSS reports, asset docs | CSS module wiring, CSS source visual rhythm, asset architecture | Do not move |
| Move `src/store/*` | HIGH / DO NOT MOVE | Store build, route derivation, manifest, JSON-LD, proof, category pages | Store tools, docs, guards, package scripts | Store proof, Store isolation, asset architecture, content source boundary | Do not move |
| Move `qa/*` | MEDIUM/HIGH | Package scripts, CI reconciliation, `ci:qa`, docs command index | `package.json`, `QA-COMMANDS.md`, CI reconciliation docs, workflows if command strings change | CI reconciliation, docs contract, repo tidy | Only under dedicated QA path task |
